# Git feature backlog

## Partial staging

- [ ] Stage individual hunks or lines.
- [ ] Unstage individual hunks or lines.

## Rebase and merge

- [ ] Add interactive rebase with squash, fixup, reorder, edit, and reword actions.
- [ ] Support squash merges.
- [ ] Support forced merge commits (`--no-ff`).
- [ ] Support custom merge strategies and strategy options.
- [ ] Support custom merge commit messages.

## Cherry-pick and revert

- [ ] Cherry-pick multiple selected commits.
- [ ] Cherry-pick commit ranges.
- [ ] Revert merge commits with a selectable mainline parent.

## Commits

- [ ] Sign commits with GPG or SSH keys.
- [ ] Add sign-off trailers to commits.
- [ ] Set an alternate commit author.
- [ ] Set custom author and commit dates.
- [ ] Create empty commits.
- [ ] Configure whether Git hooks run for a commit.

## Stashes

- [ ] Stash selected files.
- [ ] Stash staged changes only.
- [ ] Stash while keeping the index.
- [ ] Create a branch from a stash.

## Remotes and refspecs

- [ ] Rename remotes.
- [ ] Configure a remote's push URL separately from its fetch URL.
- [ ] Support custom fetch and push refspecs.
- [ ] Support mirror fetches and pushes.
- [ ] Push all branches.
- [ ] Push all tags.

## Tags

- [ ] Create signed tags.
- [ ] Force-update or move existing tags.

## History and diagnostics

- [ ] Add file and line blame.
- [ ] Add commit bisection workflows.
- [ ] Manage Git notes.
- [ ] Manage rerere conflict resolutions.

## Patch and repository utilities

- [ ] Create and apply patches.
- [ ] Support email patch workflows (`format-patch` and `am`).
- [ ] Create repository archives.
- [ ] Create and import Git bundles.

## Repository structures

- [ ] Manage submodules.
- [ ] Create, list, and remove worktrees.
- [ ] Manage sparse checkouts.
- [ ] Manage Git LFS installation, tracking rules, and objects.

## Hosting integrations

- [ ] Manage pull requests and reviews.
- [ ] Manage issues.
- [ ] Manage releases.
- [ ] Add broader GitHub hosting and authentication workflows.
